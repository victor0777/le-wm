## Task 7: L2 VP Auxiliary Supervision 구현 및 학습

- **시도**:

  - **zeroed motion이 더 이상 correct보다 좋지 않음** — VP supervision이 motion 활용 개선
  - 방향이 올바름: 구조적 이해가 motion conditioning을 간접적으로 강화
- **소요**: ~3시간 (구현 + 디버깅 + 학습 + 평가)

## Task 8: L2 Ablation (Lane-only vs Depth-only) + ADR-006
- **목표**: L2에서 motion conditioning 개선의 원인 분리
- **시도**: Lane-only (depth_weight=0)와 Depth-only (lane_weight=0) 병렬 학습 (GPU 0, 1)
- **결과**:
  - Lane-only: shuffled +4.8%, zeroed -0.3% → 차선만으로는 motion 개선 안 됨
  - Depth-only: shuffled +9.7%, zeroed +2.4% → L2 전체(+10.0%, +2.3%)와 거의 동일
  - **결론: Depth가 motion conditioning의 핵심 원인** (lane은 기여 없음)
  - Codex debate (ADR-005) branching: "depth가 핵심 → L3 즉시 진행"
- **소요**: ~1.5시간 (학습 병렬), ~5분 (평가)
- **다음**: ADR-006 — VP depth를 OccAny + LiDAR로 교체, OccAny 프로젝트에 RTB inference 요청

## Task 4: ADR-002 Experiment Matrix 실행
- **목표**: horizon과 Δψ가 motion conditioning에 미치는 영향 비교
- **시도**: A3(3D,pred=4), B1(4D,pred=1), B3(4D,pred=4) 순차 학습 (각 10 epochs, single A100)
- **결과**:

  | Exp | Action | num_preds | Correct MSE | Shuffled Gap | Zeroed Gap |
  |-----|--------|-----------|-------------|-------------|-----------|
  | A1 | 3D | 1 | 0.0281 | +8.3% | -1.9% |
  | A3 | 3D | 4 | 0.0533 | +6.5% | -5.0% |
  | B1 | 4D+Δψ | 1 | 0.0323 | +8.3% | -1.1% |
  | B3 | 4D+Δψ | 4 | 0.0759 | +3.9% | -5.8% |

  **핵심 발견**:
  - Horizon 증가(A3,B3): MSE 상승(예측 어려워짐)하나 motion gap 개선 안 됨, 오히려 악화
  - Δψ 추가(B1,B3): motion gap에 유의미한 차이 없음
  - 모든 실험에서 zeroed가 correct보다 나음 → predictor가 action을 noise로 취급
  - **결론**: feature/horizon 변경으로는 motion conditioning 강화 불가. 아키텍처 수준 변경 필요

- **소요**: ~7.5시간 (학습), ~15분 (평가)
- **다음**: 방향 재설정 — visual representation 기반 anomaly detection으로 전환, 또는 state encoder/pixel masking 등 아키텍처 변경

## Task 5: 사고 영상 Surprise 분석
- **목표**: 정상 주행으로 학습한 world model이 사고 영상에서 "surprise" 신호를 보이는지 검증
- **시도**: accident_analysis 라벨 (158 accident, 30 non_accident) 활용. 60개 사고 + 15개 비사고 mp4에서 프레임 추출 → 모델로 frame별 surprise score 계산 → 사고 시점(time_sec) 전후 패턴 분석
- **결과**:
  - 사고 영상 평균 surprise 5.74, 비사고 4.24 → **1.35x 비율** (사고가 더 높음)
  - 그러나 사고 직전 2초 surprise 상승 median 0.96x → 일관된 pre-crash signal 약함
  - 40%만 사고 전 surprise 상승, 60%는 상승 없음
  - 원인: RTB 주행 데이터 vs 블랙박스 사고 영상의 domain gap (화각, 화질, 환경 다름)
- **소요**: ~10분 (구현 + 75개 영상 처리)
- **다음**: domain gap 축소 (fine-tuning), surprise 변화율 기반 탐지, ROC curve 분석
